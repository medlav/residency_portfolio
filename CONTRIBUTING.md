# Contributing to Residency Portfolio Manager

Thank you for your interest in contributing.

This is a personal open-source project developed by **medlav**, primarily to address practical needs within Italian medical residency training. Contributions are welcome, but the scope and direction remain guided by the original intent of the project.

---

## 1. Guiding Principles

This project aims to be:

* Local-first
* Privacy-respecting
* Simple and maintainable
* Aligned with Italian residency documentation structures
* Technically transparent

It is not intended to become enterprise software or a large-scale platform.

---

## 2. Types of Contributions

Contributions may include:

* Bug reports
* Code refactoring
* UI/UX improvements
* Documentation improvements
* Test expansion
* Adaptation for other specialties
* Improvements aligned with Italian normative requirements

Feature proposals should remain consistent with the project's scope (offline, local-first, lightweight).

---

## 3. How to Contribute

1. Fork the repository.
2. Create a descriptive feature branch.
3. Commit changes with clear messages.
4. Submit a Pull Request with:

   * A concise explanation
   * Rationale for the change
   * Any relevant normative references (if applicable)

Small, focused pull requests are preferred.

---

## 4. Code Style

* Follow standard Flutter/Dart conventions.
* Prefer readability over abstraction.
* Avoid unnecessary dependencies.
* Maintain compatibility with stable Flutter releases.

---

## 5. Normative References

If contributing changes related to Italian residency requirements:

* Reference the relevant decree or framework (e.g., DM 68/2015).
* Avoid hardcoding assumptions that may change legally.
* Clearly document the rationale in comments.

The project does not claim official regulatory validation.

---

## 6. Review Process

As this is a personal project:

* Reviews may take time.
* Not all proposed changes may be accepted.
* Architectural direction remains at the discretion of the maintainer.

Respectful and constructive discussion is encouraged.

---

## 7. Code of Conduct

Contributors are expected to:

* Communicate respectfully
* Avoid personal attacks
* Provide constructive feedback
* Focus on technical and structural improvements

---

Thank you for contributing to an open, privacy-respecting tool.
